Project Overview

Microfluidic Bacterial Analysis System

The Microfluidic Bacterial Analysis System is designed to detect bacterial growth using OD600 sensors. This system provides real-time monitoring and graphing of bacterial growth over time. Our goal is to develop a "Mini Lab" microfluidic system that operates automatically in response to bacterial growth. The system features modular parts that are easy to replace. Supported by a real-time website, it allows for remote control and monitoring of operations using an STM32F446RE microcontroller with FreeRTOS for efficient multitasking.

Product Capabilities

  • 1. OD600 Sensor Monitoring: Monitors bacterial growth density.
  • 2. Temperature Control: Maintains optimal conditions for bacterial growth.
  • 3. Pump Control: Automates the transfer of bacterial cultures.
  • 4. Web-Based Dashboard: Provides a user-friendly interface for data reading and sensor testing.

System Components

  • - OD600 Sensors: Measure bacterial growth in lab tubes and mating chambers.
  • - Temperature Sensors and Heating Surface: Ensure optimal growth conditions.
  • - Pumps: Automate air insertion, bacterial mixing, and cleaning processes.
  • - Microcontroller (STM32F446RE): Controls sensors and actuators, running FreeRTOS for multitasking.
  • - ESP32: Hosts the real-time website for monitoring and control.

Electrical Scheme

  • - Microcontroller: STM32F446RE for controlling sensors and actuators.
  • - Power Supply: USB charger for STM32 and ESP32, 230V to 5V adapter for pumps, 230V to 12V adapter for heating and fans.
  • - Sensors and Actuators: Connected to the microcontroller to ensure synchronized operation.

Software and Firmware

  • - Control Algorithms: Implemented in the STM32F446RE microcontroller for sensor readings, actuator operations, and data communication.
  • - Real-Time Website: Hosted on ESP32, features a user-friendly interface for real-time monitoring and control of the system.

Future Work and Risk Management

  • - Additional Experiments: Gather more data, refine sensor calibration, and enhance experimental setups.
  • - System Optimization: Improve control algorithms, enhance user interface, and ensure seamless integration of components.
  • - Risk Management: Address potential contamination, integration issues, and sensor calibration challenges.

Website Simulation Features

  • - DashBoard Simulation: System Diagram: Visual representation of the system setup and components.
  • - DashBoard Simulation: Data Monitoring: Real-time graphs for OD600 and temperature values.
  • - Task Editor Simulation: Task Management: Choose a sensor to set tasks and schedules. Manage multiple sensors and set OD600 values for automatic actions.
  • - Settings Simulation: PWM Adjustments: PWM1 adjusts Pump 1, Pump 2, and Pump 4. PWM2 modifies Pump 3, which is responsible for mixing. When V01 or V02 is set to -1, you will see the value of the ADC instead of the OD600 to get the maximum value of V01 or V02.

Experiment Examples

  • 1. Precision Growth Measurements and Automation:
    • Goal: Quantify bacterial growth using OD600 sensors and automate culture transfer.
    • Steps: Calibration with water and diluted coffee, pump activation based on OD600 readings.
  • 2. System Functionality and Component Validation:
    • Goal: Validate system power requirements, mixing behavior, and temperature control.
    • Steps: Measure power consumption, verify pump and heating surface functionality, and automate data recording.
